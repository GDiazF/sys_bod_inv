Reglas de uso
1. Tokens primero — nunca hardcodear colores en pantallas; usar clases Tailwind del preset.

2. ui/ vs patterns/ — átomos sin lógica de dominio; patrones combinan átomos (DataTable = Table + Pagination + scroll).

3. FormField obligatorio — todo input en pantalla va dentro de FormField con label, help y error.

4. cn() siempre — merge de clases con variantes CVA; no concatenar strings manualmente.

5. Scroll interno — TableShell, Drawer body y paneles largos usan overflow controlado; body nunca scrollea.
Anti-duplicación
Variantes con CVA — Button, Badge, Alert comparten patrón buttonVariants().

Composición — SearchInput extiende Input; QuantityInput extiende NumberInput + stepper.

Estilos compartidos — inputBase en tokens para Input, Select, Textarea.

No copiar CSS — si un estilo se repite 2+ veces, subir a ui/ o tokens.
