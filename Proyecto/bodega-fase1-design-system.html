Decisiones arquitectónicas
  • Identidad industrial premium — base luminosa tipo command center, rejilla operativa sutil, rails steel en navegación, verde reservado para acciones y estados.
  • Layout operativo full-width — el contenido principal ocupa todo el ancho disponible; solo el texto introductorio limita legibilidad (~68ch).
  • Shell fijo 100vh — el scroll vive en paneles internos, nunca en el body.
  • Tokens en Tailwind extend — una sola fuente de verdad en tailwind.config.ts.
  • Componentes atómicos en ui/ — patrones compuestos en patterns/.
  • Convención PascalCase — archivos de componente = nombre del export.
  • Utilidad cn() — merge de clases con clsx + tailwind-merge.
Listo para Fase 2 Una vez aprobada esta base, se pueden construir módulos funcionales (dashboard, movimientos, recepción, despacho) reutilizando AppShell, TableShell, FilterBar y formularios sin duplicar estilos.

ARCH · Estructura

Arquitectura de carpetas

Estructura escalable para React + TypeScript. Separación clara entre átomos, layout y patrones compuestos.

src/ ├── app/ # Rutas y layouts (Next.js / React Router) │ ├── layout.tsx │ └── providers.tsx ├── components/ │ ├── ui/ # Átomos: Button, Input, Badge, Modal… │ ├── layout/ # AppShell, Sidebar, Topbar, PageHeader │ └── patterns/ # TableShell, FilterBar, StatCard, FormSection ├── config/ │ └── theme/ │ ├── tokens.ts # Tokens tipados (const assertions) │ └── tailwind.preset.ts ├── hooks/ # useMediaQuery, useScrollLock… ├── lib/ │ └── cn.ts # clsx + tailwind-merge ├── styles/ │ └── globals.css # @tailwind + CSS vars ├── types/ # Tipos compartidos UI └── utils/ # Convención de nombres # · Componentes: PascalCase → Button.tsx exporta Button # · Hooks: camelCase con prefijo use → useBreakpoint.ts # · Utilidades: camelCase → formatSku.ts # · Tipos: PascalCase + sufijo Props/Config → ButtonProps
Patrón AppShell
AppShell envuelve toda la app:

· Sidebar fija (240px desktop)
· Topbar 56px sticky
· main con flex-1 min-h-0 overflow-auto
· Contenido sin max-width restrictivo en desktop
· Body nunca hace scroll
Patrón TableShell
Contenedor con altura fija o max-h:

· Header de tabla sticky top-0
· Body en overflow-y-auto
· Toolbar de filtros fuera del scroll
OPS · Layout

Responsive y scroll interno

Desktop como superficie principal. Tablet y móvil reutilizan los mismos componentes con adaptaciones de layout.

Desktop ≥ 1280px Contenido a ancho completo con gutters amplios. Sidebar 256px. Tablas y splits ocupan todo el espacio horizontal; scroll solo en paneles internos.
Tablet 768–1279px Sidebar colapsable a iconos. Grids en 2 columnas. Filtros en fila compacta. Tablas mantienen scroll interno.
Móvil < 768px Sidebar en drawer. Stats y grids apilados. Tablas con scroll horizontal + vertical en contenedor.

Reglas CSS obligatorias
